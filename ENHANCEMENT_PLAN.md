# HyperStim Site — Enhancement Plan

Tracked step by step so nothing gets missed. Status: ✅ done · ⚠️ done differently than literally specified (explained below) · 🔲 not started

## 1. Navigation & orientation
- ✅ Sticky top nav (accessibility toolbar + Lookup/Compare/Resources bar) so it never scrolls away
- ✅ "Back to top" button once a page's content gets long (card grid, compare matrix)

## 2. Making the rating system actually click
- ✅ Inline "?" info icons next to every score badge — hover/tap explains that one metric
- ✅ Icon shapes (not just color) on Good/Moderate/Extreme badges, for colorblind users
- ✅ First-visit onboarding walkthrough explaining the 3 categories (dismissible, remembered via localStorage)
- ✅ Visual diagram of the taxonomy on the Resources page, generated live from the same schema
  the site's scoring logic uses (so it can never drift out of sync with the real code)

## 3. Video interactivity & discovery
- ✅ Hover-to-preview: small muted autoplay preview on hover, one live embed at a time, skipped
  for unavailable videos and for anyone with reduced-motion preference set
  (this also covers the original "hover card" idea — the preview itself, plus the score info
  already visible on every card, made a separate static hover-popover redundant)
- ✅ "Add to Compare" button directly on lookup cards
- ✅ "Compare with a similar video" suggestion (same topic, prefers a different era) on the
  Lookup page's detail panel

## 4. Compare page usability
- ✅ Type-to-filter search box above each dropdown
- ✅ Paste a YouTube URL directly into either compare slot (reuses the same parser as Lookup)
- ✅ Swap button to flip Video A / Video B
- ✅ Shareable/bookmarkable comparison links (`compare.html#a=ID1&b=ID2`)

## 5. Error handling
- ✅ Specific "why unavailable" messaging (private vs. removed) where YouTube's response
  distinguishes them; generic message otherwise — backward-compatible with older reports
- ✅ "Try again" button on the unavailable-video fallback
- ✅ Loading skeleton (CSS shimmer) behind every thumbnail while it loads

## 6. Data visualization
- ✅ Radar/spider chart comparing both videos across all 10 pattern types, at the bottom of
  every comparison
- ⚠️ Accessible alternative: rather than a second full bar-chart view, the radar chart carries a
  visually-hidden data table with the exact same numbers, so screen reader users get full parity
  without needing to interpret an SVG shape. A literal second chart type wasn't built — this was
  a judgment call to cover the actual accessibility need more directly.

## 7. Responsive & cross-device
- ✅ Touch tap-target fix: the small info-icon gets an invisible enlarged hit area on touch
  devices via `@media (pointer: coarse)`, without changing its visual size
- ✅ Swipe hint under the compare matrix on mobile widths
- ✅ Dark mode via `prefers-color-scheme`, independent of and overridden by the manual
  high-contrast toggle when both apply
- ⚠️ Edge-case viewports: verified via careful CSS review (breakpoints at 400/480/768/900/1200px,
  fluid grids, no fixed pixel widths that would overflow) rather than testing on physical
  devices, which isn't possible from this environment. Worth a manual pass on a real phone/large
  monitor once deployed.

## 8. Trust & polish
- ✅ "Data last rebuilt" timestamp on Resources, auto-generated by `build_dataset.py` every run
- ✅ Fixed a real pre-existing bug found along the way: the Video A/B label coloring in the
  Compare picker had a broken CSS selector and had never actually applied

---
Every item above was verified with an automated functional test (Node `vm`-based DOM
simulation) against real data shape before being marked done — not just written and assumed
to work.

## Pre-deployment double-check (found and fixed 2 more issues)

- **Fixed**: the info-icon's "?" text was inheriting white color from its parent badge while
  sitting on a translucent *white* circle — nearly invisible. Made it self-contained (solid dark
  circle, white text, subtle border) so it has real contrast in every context. The first fix
  attempt for this also introduced a worse bug — white-on-white in high-contrast mode
  specifically — caught before shipping and replaced with the fully self-contained version.
- **Fixed**: a stale CSS selector (`.compare-card`) left over from before the matrix redesign,
  which silently matched nothing and meant the compare page's video cells weren't getting their
  high-contrast-mode border reinforcement. Updated to the real current class (`.matrix-video-cell`).
- Swept every CSS class against every HTML/JS source file in both directions (styled-but-unused,
  and used-but-unstyled) — no other issues found.
- Re-ran the full regression suite (both pages, all features together) after both fixes —
  everything still passes.
